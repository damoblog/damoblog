Puerts配置vscode环境

发表于2021-08-25|更新于2021-08-26|技术Unity
|阅读量:

准备工作

  • 安装vscode

  • 安装Node.js

  • 安装npm, npm为Node.js 自带的包管理工具，上面安装Node时就已经安装上了

  • 在 项目/Assets/ 目录下建TypeScript文件夹用于存放项目typescript代码

    • 安装TypeScript到系统环境（在 项目/Assets/TypeScript/目录 ）下打开控制台（下同））
      • 输入 npm install typescript -g 安装
    • 安装node.js依赖模块
      • 输入 npm install

  • tsc编译器

    • 重启vscode
    • 在vscode控制台输入 tsc
    • 运行失败的话，按照以下步骤操作(windows)
      • 以管理员身份运行 Windows Power Shell 终端
      • 执行命令: set-ExecutionPolicy RemoteSigned
      • 在控制台提示下输入 A 并回车

  • Unity安装VscodePackage(Package Manager中搜索Visual Studio Code Editor即可)

  • 设置编辑器为 vscode

    • windows
      • Edit -> Prefereneces… -> External Tools -> External Script Editor
        • 选择 Visual Studio Code
    • Mac
      • Unity -> Preferences…-> External Tools -> External Script Editor
        • 选择 Visual Studio Code

  • 修改VS Code Path

    • Edit -> Prefereneces… -> vscode
      • 修改 VS Code Path为正确地址(一般为安装目录/bin/code.cmd)

  • 安装插件 vscode扩展中安装

    • C#
    • Auto-Using for C#
    • Chinese (Simplified) Language Pack for Visual Studio Code
    • Debugger for Unity
    • Unity Code Snippets
    • Unity Snippets
    • Unity Tools

vscode调试 C# 代码

配置launch.json

删除项目中.vscode目录下launch.json文件，将右边选项切换到运行和调试目录，点击创建launch.json，选择unity Debugger选项
这里有个坑点，有时候可能没有这个选项，选项切换到插件(扩展)目录点击Debugger for Unity打开扩展Debugger for Unity界面，再将选项切回运行和调试目录, 点击创建launch.json选择unity Debugger选项
此时运行unity代码即可在vscode中调试C#代码

C#代码代码没有跳转和自动提示功能

按下键盘的Ctrl/Command + Shift+ P快捷键，选定工程目录下的sln文件。
稍等几秒等待自动加载完成以后可以重试一下是否有代码提示功能。

配置ts->js环境

修改package.json并安装依赖

  • name，项目名称（用于发布到npm仓库上的），在游戏开发中用不上
  • version，项目版本（用于发布到npm仓库上的），在游戏开发中用不上
  • scripts，定义的构建脚本，可以将任意工作流定义其中。npm run 任务名 来执行某个任务
  • dependencies，这里定义了项目中要用到的依赖
    • @swc-node/core，swc 编译器
    • fs-extra，一个 node 文件操作模块增强版本
    • uglify-js，一个 Js 代码压缩模块
      在项目目录下运行 npm install 来拉取依赖到本地，拉取到的所有的依赖都将保存在 node_modules 目录中

修改tsconfig.json 文件

  • target，编译最高可支持的语法。
  • module，模块规范，Puerts 默认只支持 CommonJs
  • jsx，看项目情况需要，可以使用 jsx 来写 ui
  • lib，内置Js API 的默认类型定义
  • inlineSourceMap，在调试时，开发工具要将 Js 代码与原始 Ts 对应上都依赖于 Map 信息。
  • moduleResolution，模块解析策略
  • experimentalDecorators，启用装饰器
  • baseUrl，基本目录
  • typeRoots，指定哪些文件默认需要引入，否则就需要手动使用『三斜杠』指令手动引入。
  • outDir，编译输出目录
